# Contribution Guidelines

Thank you for considering contributing to the Thamer Immortal Protocol! We welcome contributions from everyone. Please follow the guidelines outlined below to ensure a smooth contribution process.

## Reporting Bugs
- **Check Existing Issues**: Before reporting a new bug, please check if the issue has already been reported.
- **Create a Bug Report**: If the issue is not mentioned, create a new issue. Include information such as:
  - A clear and descriptive title
  - Steps to reproduce the issue
  - Expected vs. actual behavior
  - Screenshots, logs, or other relevant data

## Suggesting Features
- **Feature Requests**: We appreciate new ideas! If you have a feature in mind, please open a new issue to discuss it. Include:
  - A detailed description of the feature
  - Use cases and benefits of the feature

## Code Contribution Process
1. **Fork the Repository**: Click on the "Fork" button at the top right of the repository page.
2. **Create a Branch**: Create a new branch for your feature or fix from the main branch. Use a descriptive branch name.
3. **Make Changes**: Implement your changes or feature.
4. **Test Your Changes**: Ensure that all existing tests pass and, if applicable, add new tests.

## Coding Standards
- Follow the project's coding style guidelines. Consistency is key for readability.
- Write clear and descriptive commit messages.
- Comment your code where necessary to explain complex logic.

## Pull Request Process
1. **Create a Pull Request**: Once you are ready to contribute, create a pull request from your branch to the main branch.
2. **Add a Description**: Provide a clear title and description of the changes made, including any relevant issue numbers.
3. **Review Feedback**: Be open to feedback and ready to make necessary adjustments based on review comments.

## Code of Conduct
We strive to make participation in our project a harassment-free experience for everyone. Please read our [Code of Conduct](link-to-code-of-conduct) to understand how we expect contributors to behave.

We appreciate your contributions and look forward to working with you!
